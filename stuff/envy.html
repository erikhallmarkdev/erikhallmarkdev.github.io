Envy

Envy is a simple Node/Value configuration language designed to make writing and reading configuration documents as fast and simple as posible. Envy offers a simple yet power syntax that is incredibly quick to learn.

At the time of writing this there is only a .NET build of Envy, but many more languages are in the works, including JavaScript, C++, and Python.

The Basics

Envy is a lightweight, text based, language independent configuration language.

Envy can hold three basic types of data Bools, Numbers, and Strings. This data is stored in a hierarchy of Items, Values, and Nodes.

Items are a single piece of data, they can be a Bool, a Number, or a String.

Values essentially arrays of items, they can store as many items as needed

Nodes are a group of values and other nodes. A node identifies the nodes and values that it holds with names, a name can be composed of letters, numbers, and underscores but the first character must be a letter. Each value in a node has a unique name, but a multiple nodes can all share the same name.

The Syntax

An item can be be one of three things

  • A bool can be written as a simple "true" or "false" in Envy.
  • A number can be any number, negative or possitive, with as many numbers after the decimal place as needed
  • A string is just texted held between two quotation marks. Note that a quotation mark will not count if it is escaped with a backslash

Example Code

{% highlight envy %} value ( "Hello, world!" ) OtherValue ( "Strings, bools, numbers, oh my!", true, false, 3.1415 ) Node { value ( "Value within node" ) #Comment NestedNode { } NestedNode { #Note that these two nodes share the same name, nodes can do that, values cannot. } } {% endhighlight %}
